Engineering log — real issues hit while shipping this

ISSUECloudFront served AccessDenied from S3 even with a valid Origin Access Control.
CAUSEThe OAC had been recreated a few times; the bucket policy still pointed at a stale distribution reference.
FIXCreated one clean OAC, let CloudFront auto-generate the matching bucket policy, then invalidated the cache.
ISSUEAttaching Lambda as an API Gateway trigger created the wrong route.
CAUSEThe "Add trigger" flow auto-names routes after the function, not the path the frontend actually calls.
FIXCreated GET /visitor manually in Routes and attached the Lambda integration directly to it.
ISSUEThe $default stage looked "undeployed" and couldn't be selected in the deploy dialog.
CAUSEAuto-deploy was already on — HTTP APIs deploy themselves on every change, so manual deploy is disabled by design.
FIXNo action needed — confirmed changes were live via a direct curl to the invoke URL instead.
