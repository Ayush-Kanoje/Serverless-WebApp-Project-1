Troubleshooting Logs

Issue: Couldn't delete CloudFront distribution after deleting the S3 bucket.

Cause: CloudFront still had the deleted S3 bucket configured as its origin.

Fix: Disabled the distribution, updated/removed the origin configuration, waited for deployment, then deleted it.

Issue: Lambda couldn't access DynamoDB

Cause: Missing IAM permissions.

Fix: Attached a least-privilege IAM policy allowing only the required DynamoDB actions.

Issue: CloudFront returned "AccessDenied" from S3 despite a valid Origin Access Control.

Cause: Bucket policy was stale after recreating the OAC multiple times.

Fix: Created one clean OAC and let CloudFront auto-generate the matching bucket policy.

Issue: Created a second API Gateway stage to "confirm" $default was deployed, ending up with two live URLs.

Cause: Misunderstood stage creation as a verification step, when it's actually a separate deployment target.

Fix: Confirmed both stages pointed to the same Lambda/DynamoDB backend via curl, then deleted the unnecessary stage and kept $default as the single source of truth.
